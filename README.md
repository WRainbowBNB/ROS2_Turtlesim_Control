### 项目综述
该项目分为两个节点，控制节点，负责pid调节和推力扭矩指令的计算和下发，物理仿真节点则实时计算小海龟当前运动状态并模拟水下环境，引入高斯噪声（模拟传感器噪声）和高斯-马尔科夫噪声（模拟水流）

### 通信说明
定速巡航和画圆轨迹作为服务，定点模式则作为动作，终端为客户端，控制节点为服务端，物理仿真节点订阅并计算应该分给小海龟的速度和角度，同时反馈当时的速度、加速度等数据（模拟dvl和imu），同时dvl和imu为贴合现实，时序应不同

### 核心算法
嵌套式pid：两个内环即扭矩环和推力环并行调节，外环位置环输出角速度（只在定点时开启），均为位置式

### 启动方法
终端1,2,3分别启动小海龟节点，控制节点，物理仿真节点（因为我还没学launch，所以节点需要挨个手动启动）
终端4发命令，终端5跑rqt调参

### 可改进之处
1. 机器人坐标系下，x轴方向，即东西方向虽无推进器，但是在速度急剧变化时，例如急转弯处，由于机器人质量无法避免，即在惯性系下即使水下阻尼大，x方向依然可能存在微小漂移，该情况需要考虑
2. 以及多传感器噪声（目前是模拟）可通过卡尔曼滤波进行降噪
3. 机器人坐标系向世界坐标系的转换，即手推当前世界坐标系逻辑有待优化，之前写的更像是哪里都是坐标原点
4. 定速巡航可引入MPC，嗯现在就学😋
5. 挨个终端开节点还是太繁琐了，快去学launch
